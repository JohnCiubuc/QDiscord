QDiscord  v0:12
An unofficial C++ and Qt wrapper for the Discord API.
Installing the library

Configuration

The library's main static configuration file is located in src/config.pri. Use this to set configuration options like debugging, the library's version, the library's name and a link to the library's repository.
Keep in mind, the library should not be in debug mode for any production code, since some of that debug information may involve private information. (The library shouldn't be in production code until version 1, for that matter)

Compiling

QtCreator

Just selecting the build target you want and hitting build should be enough.

Command-line

The following commands should build the project for Linux:

mkdir build
cd build
qmake -qt=qt5 ../QDiscord/QDiscord.pro
make all

Tests

If you select the Debug build target, tests will be compiled. Execute the resulting executables to run the tests, or use QtCreator's plugin to run them automatically.

Installing

Change your current working directory to the build directory and call make install.
This will install the library into /usr by default. (/usr/lib and /usr/include). To override this, call the command like this:

PREFIX=/ make install

Compiling the documentation

Make has an extra target named docs for generating the documentation.
If you wish to generate the documentation when compiling from QtCreator, add a custom build step for make with the Doxygen file's directory as the working directory and with the following arguments for make:

docs --makefile=%{buildDir}/Makefile
